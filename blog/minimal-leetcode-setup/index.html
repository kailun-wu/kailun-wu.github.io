Minimal LeetCode Setup

Oct 30th, 2014

LeetCode Online Judge is a website that hosts algorithm problems and verifies your solutions. Many of my classmates are working hard on it. While far from finishing, I really enjoyed my clean setup.

The way to do it is having two terminal windows opened side by side. On the left, mkdir a new folder to hold all *.java and *.class. Each problem requires exactly one file so I can stay away from any IDE, which is neat. Then in the right terminal window, say, vim MaxProductSubarray.java to start solving a problem:

  1. Write the skeleton code.

     import java.util.Arrays;

 // https://oj.leetcode.com/problems/maximum-product-subarray/

 public class MaxProductSubarray {
     public static void main(String[] args) {
         show(new int[]{5});
         show(new int[]{-1,3});
         show(new int[]{2,3,-2,4});
         show(new int[]{5,2,0,3,9,0});
         show(new int[]{9,2,4,-3,8,-2,-5});
     }       
     public static int maxProduct(int[] A) {
         return 0;
     }
     public static void show(int[] A) {
         System.out.println(Arrays.toString(A) + 
             " -> " + maxProduct(A));
     }
 }

  2. Before thinking about solutions, write tests in main(), for I already know the interface of solution(). Tests drive you to cover all the edge cases you’d normally overlooked when implementing solutions. Tests inspire me at times.

  3. Solve the problem by hand, code the idea and run tests.
  4. Take down notes and remember handy Java methods.

When done, in the left terminal use javac MaxProductSubarray.java to compile and java MaxProductSubarray to execute. ⌘ + 1/2 will also do the switching between terminal windows.

This setup becomes more efficient when I’m better at Vim. Vim has a seriously steep learning curve but Learn Vim Progressively (Chinese version here) made me pretty adept at it.
